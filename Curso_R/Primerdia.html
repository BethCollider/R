¿Qué es una base de datos?

Tipos de variables

Conocer los tipos de variables es esencial para capturar, limpiar, verificar y procesar bases de datos.

  • Númerico

    • Discreto (Integer) 🔢

    • Continuo (Double) 📏

  • Carácter (Character)

    • Nominal 🙍 🙍‍♂️ 👤(Factor)

    • Ordinal 🥇🥈🥉 (Factor, levels)

  • Lógico (Logic)

    • Verdadero 🙆

    • Falso 🙅‍♂️

  • Not Available (NA)

Numeric

Vamos a crear un vector númerico

numero <- 1.25

R le otorga el tipo de vector de manera automática

class(numero)
[1] "numeric"
typeof(numero)
[1] "double"

Pregunta #1

¿Qué tipo de vector es la siguiente expresión?

x <- 2L
typeof(x)
[1] "integer"

La letra “L” indica que es un valor de tipo discreto.

¿Qué tipo de vector es la siguiente expresión?

x <- 2.5L
typeof(x)
[1] "double"

R lo modifica a continuo por que los valores discretos no pueden contener decimales.

Numeric

Un vector puede contener varios elementos el emplear la función c()

numeros <- c(1.25, 3.5, 2.3, 9.7, 5L)

Recuerda, R va a otorgar el tipo de vector de manera automática.

typeof(numeros)
[1] "double"
print(numeros)
[1] 1.25 3.50 2.30 9.70 5.00

Pregunta #2

¿Qué tipo de vector es la siguiente expresión?

numeros <- c(1L, 3L, 2L, 9.7, 5L)
typeof(numeros)
[1] "double"
print(numeros)
[1] 1.0 3.0 2.0 9.7 5.0

Un valor continuo no puede ser discreto, pero un valor discreto si puede ser continuo.

Character (Nominal)

Vamos a crear un vector tipo carácter

caracter <- "Jaime"
is.numeric(caracter)
[1] FALSE
is.character(caracter)
[1] TRUE

Es necesario ingresar los información entre ” “ para indicar que es un vector de tipo carácter.

caracter2 <- Roberto
Error in eval(expr, envir, enclos): objeto 'Roberto' no encontrado

Character (Nominal)

Un vector puede contener varios elementos el emplear la función c()

Nombres <- c("Pedro", "Veronica", "Jaime", "pedro", "Juan")
class(Nombres)
[1] "character"
typeof(Nombres)
[1] "character"

¿Cúantos nombres distintos hay en el vector Nombres?

dplyr::n_distinct(Nombres)
[1] 5

R reconoce mayúsculas y minúsculas

Pregunta #3

¿Cúantos elementos únicos tiene el siguiente vector?

Nombres <- c("Pedro", "Veronica ", " Juan ", "Jaime ", "Pedro ", " Juan")
dplyr::n_distinct(Nombres)
[1] 6

R reconoce los espacios. Por lo tanto, hay 6 valores únicos.

Eliminar espacios

R tiene varias funciones que pueden ayudarnos a corregir dichos errores.

Nombres <- stringr::str_trim(Nombres)
print(Nombres)
[1] "Pedro"    "Veronica" "Juan"     "Jaime"    "Pedro"    "Juan"    
dplyr::n_distinct(Nombres)
[1] 4

Character (ordinal)

En algunas ocaciones, nuestros datos representan un orden

Salud <- c("Mala", "Buena", "Muy mala", "Regular", "Muy buena")
sort(Salud)
[1] "Buena"     "Mala"      "Muy buena" "Muy mala"  "Regular"

En estos casos, debemos indicarle a R dicho orden.

Salud <- factor(Salud, levels = c("Muy buena", "Buena", "Regular", "Mala", "Muy mala"))
class(Salud)
[1] "factor"
sort(Salud)
[1] Muy buena Buena     Regular   Mala      Muy mala 
Levels: Muy buena Buena Regular Mala Muy mala
typeof(Salud)
[1] "integer"
as.integer(Salud)
[1] 4 2 5 3 1

Altura <- c("Baja", "Muy Baja", "Media", "Alta", "Muy Alta",

Altura <- factor(Altura, levels = c("Extremadamente baja", "Muy Bajo", "Baja",
                                    "Media", "Alta", "Muy Alta", "Extremadamente alta"))

Not available (NA) 🤯

Valores con NA complican el manejo de bases de datos.

vectorConNA <- c(1,2,3,4,5,6,NA)

  • No puede ser ordenado

    sort(vectorConNA)
    [1] 1 2 3 4 5 6

  • No puede ser filtrado de manera indirecta

    vectorConNA[vectorConNA>2]
    [1]  3  4  5  6 NA

  • Muchas funciones no aceptan valores con NA

    sum(vectorConNA)
    [1] NA

¿Qué hacer con los NA?

  • Elminarlos ❌

  • Imputarlos

    Peso <- c(70,75,NA,85,90)
mean(Peso, na.rm = T)
    [1] 80

  • Interpolarlos

    Altura <- c(170,175,173,185,190)
lm(Peso~Altura)
    
Call:
lm(formula = Peso ~ Altura)

Coefficients:
(Intercept)       Altura  
       -100            1

Data frame (bases de datos)

  • Es el objeto más habitual para almacenar datos en R.

  • Cada fila representa un individuo u observación.

  • Cada columna representa una cualidad del individuo.

data.frame(Nombre=c("Pedro", "Karla","Jaun", "Raul", "Sandra"),
              Edad=c(25,22,33,22,26))
  Nombre Edad
1  Pedro   25
2  Karla   22
3   Jaun   33
4   Raul   22
5 Sandra   26

¡Cuidado!

Un data frame siempre debe tener el mismo número de valores por columna (tibble()).

dplyr::tibble(Nombre=c("Pedro", "Karla","Jaun", "Raul", "Sandra"),
              Edad=c(25,22,33,22))
Error:
! Tibble columns must have compatible sizes.
• Size 5: Existing data.
• Size 4: Column `Edad`.
ℹ Only values of size one are recycled.

¿Qué hacer?

En estos casos, se puede indicar que la edad de Sandra es desconocida (NA) 🤷

dplyr::tibble(Nombre=c("Pedro", "Karla","Jaun", "Raul", "Sandra"),
              Edad=c(25,22,33,22, NA))
# A tibble: 5 × 2
  Nombre  Edad
  <chr>  <dbl>
1 Pedro     25
2 Karla     22
3 Jaun      33
4 Raul      22
5 Sandra    NA
